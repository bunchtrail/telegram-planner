# Habits iOS Focus List Design

**Date:** 2026-04-18

## Goal

Упростить мобильный iOS-экран привычек до сценария "открыл и быстро отметил на сегодня", убрав тяжёлую верхнюю панель с дублирующим summary и датой.

## Problem

Текущий `MobileHabitsTab` тратит первый экран на крупную `SurfaceCard` с несколькими уровнями метаданных:

- заголовок раздела,
- количество привычек,
- статус по выбранной дате,
- недельный total,
- отдельный chip с датой.

Для habits это создаёт лишнюю визуальную иерархию. Пользователь приходит не читать summary, а быстро взаимодействовать со списком привычек. На iPhone экран ощущается как dashboard, хотя по задаче это daily checklist.

## Decision

Для mobile/iOS экран привычек переходит в режим `Focus List`.

- Экран якорится на текущем дне и текущей неделе, а не на `selectedDate` из общего planner state.
- Верхний блок сокращается до компактной шапки: `Привычки` + кнопка `Добавить`.
- Под шапкой остаётся только компактная строка прогресса в виде pills.
- Основной список привычек начинается сразу после этой строки.
- Внутри каждой привычки сохраняется недельная сетка как вторичный контекст.

## UI Structure

### Top row

- Заголовок `Привычки`
- Кнопка `Добавить`
- Без крупной card-обёртки
- Без отдельного заголовка про "сегодня" и без даты в hero-блоке

### Progress pills

Компактная строка сразу под шапкой:

- `X из Y выполнено`
- `N отметок за неделю`

Эти метрики дают ежедневный и недельный сигнал, но не конкурируют со списком за внимание.

### Habit list

- Неполные привычки идут первыми
- Сортировка определяется состоянием на сегодня
- Внутри карточки остаются:
  - иконка,
  - имя,
  - компактный weekly count,
  - `HabitWeekGrid`

## Behavior

- `MobileHabitsTab` использует текущую дату (`new Date()`) как источник истины для:
  - выбранного дня,
  - недельного диапазона,
  - daily summary,
  - порядка привычек.
- Проп `selectedDate` сохраняется ради совместимости интерфейса, но не управляет mobile habits UI.
- `HabitWeekGrid` не меняется по контракту: будущие дни по-прежнему disabled, today подсвечен, completed/pending состояния сохраняются.
- Bottom sheet добавления привычки и двухшаговое удаление из меню не меняются.

## Non-Goals

- Не добавляем новую бизнес-метрику streak, потому что для habits она сейчас не присутствует в данных/контрактах.
- Не меняем desktop habits layout.
- Не трогаем optimistic/realtime логику и структуру данных.

## Validation

Изменение считается корректным, если:

- mobile habits screen больше не показывает крупную summary-card сверху;
- экран визуально стартует со списка привычек;
- summary остаётся компактным и не привязан к `selectedDate`;
- `MobileHabitsTab` тестами подтверждает today-first сортировку и summary;
- add/delete/haptic interactions не регрессируют.
